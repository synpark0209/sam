# 시스템 구현 문서

> 최종 업데이트: 2026-04-05

## 1. 아키텍처

### 1.1 전체 구조
```
client/          → Phaser 3 + React + TypeScript (Vite)
server/          → NestJS + TypeORM + PostgreSQL
shared/          → 클라이언트/서버 공유 타입 및 데이터
```

### 1.2 씬 흐름
```
TitleScene → LobbyScene ─→ WorldMapScene → [게스트선택] → DialogueScene → BattleScene
                         ─→ PvPArenaScene (배치/자동전투)
                         ─→ DailyDungeonScene (웨이브 전투)
                         ─→ 가챠 (LobbyScene 내)
                         ─→ 상점 (LobbyScene 내)
                         ─→ 장수관리 (LobbyScene 내)
                         ─→ 일일 임무 (LobbyScene 내)
                         ─→ RankingScene
```

### 1.3 카메라 시스템
- **메인 카메라**: 줌/드래그 스크롤 (전투 맵)
- **UI 카메라**: 줌 1x, 스크롤 0 (턴 정보, 버튼, 메뉴, 유닛 정보 패널)

### 1.4 서버 권위적 시스템
- **가챠**: 서버에서 확률 계산, 재화 차감, 장수 추가 (POST /gacha/pull)
- **세이브**: 서버 DB 저장 (campaignProgress JSONB)
- **인증**: JWT + Telegram initData 검증
