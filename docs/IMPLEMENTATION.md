# 시스템 구현 문서

> 최종 업데이트: 2026-03-26

## 1. 아키텍처

### 1.1 전체 구조
```
client/          → Phaser 3 + React + TypeScript (Vite)
server/          → NestJS + TypeORM + PostgreSQL
shared/          → 클라이언트/서버 공유 타입 및 데이터
```

### 1.2 씬 흐름
```
TitleScene → LobbyScene ─→ WorldMapScene → [게스트선택] → DialogueScene → BattleScene
                         ─→ BattleScene (자유전투)
                         ─→ 가챠 (LobbyScene 내)
                         ─→ 장수관리 (LobbyScene 내)
                         ─→ RankingScene
```

### 1.3 카메라 시스템
- **메인 카메라**: 줌/드래그 스크롤 (전투 맵)
- **UI 카메라**: 줌 1x, 스크롤 0 (턴 정보, 버튼, 메뉴, 유닛 정보 패널)

### 1.4 서버 권위적 시스템
- **가챠**: 서버에서 확률 계산, 재화 차감, 장수 추가 (POST /gacha/pull)
- **세이브**: 서버 DB 저장 (campaignProgress JSONB)
- **인증**: JWT + Telegram initData 검증

| 엔티티 | 주요 필드 |
|--------|----------|
| User | id, username, passwordHash, telegramId |

### 6.2 shared/ 데이터 파일
| 파일 | 내용 |
|------|------|
| `data/skillDefs.ts` | 공용 10종 + 병종 기본 18종 + 고유 19종 = 47종 |
| `data/classSkillDefs.ts` | 병종별 기본 스킬 3단계 진화 |

| `data/promotionDefs.ts` | 승급 경로 6병종 x 2단계 |
| `data/unitClassDefs.ts` | 병종 특성 (지형비용, 8방향 공격) |
| `data/pvpDefs.ts` | PvP 티어(5단계)/ELO/시즌보상/일일티켓 |
| `data/equipmentDefs.ts` | 장비 정의 |
| `data/campaign/*.ts` | 프롤로그 + 챕터 1~2 시나리오 |
