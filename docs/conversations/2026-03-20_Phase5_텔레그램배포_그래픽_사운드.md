### 2. Phase 5 - 텔레그램 미니앱 배포
- **배포 인프라 구축**: Vercel(프론트) + Railway(백엔드+PostgreSQL)
- **클라이언트**: API URL 환경변수화 (`VITE_API_URL`), `.env.example` 생성
- **서버**: CORS 설정 개선, DB synchronize 환경변수 제어, Dockerfile 작성
- **텔레그램 통합**:
  - 서버: `POST /auth/telegram` 엔드포인트 (initData HMAC 검증)
  - User 엔티티에 `telegramId` 필드 추가
  - 클라이언트: Telegram WebApp SDK 연동, TitleScene에서 자동 로그인
  - BotFather `/newapp` 으로 미니앱 등록 (Menu Button은 initData 미전달)

### 4. 그래픽 고도화
- SpriteGenerator 96x96 해상도로 업그레이드 (48x48 축소 렌더링)
- UnitSpriteManager: 외부 PNG 이미지 로딩 시스템 (폴백: 프로시저럴)
- AI 생성 유닛 이미지 6병종 x 2진영 = 12개 적용 (ChatGPT DALL-E)
  - infantry, cavalry, archer, strategist, bandit, martial_artist
  - cavalry_archer 이미지도 준비됨 (병종 미추가)

### 5. 모바일 UI 최적화
- **카메라 줌**: 화면 너비에 따라 가로 ~7칸 보이도록 자동 줌
- **드래그 스크롤**: 터치 드래그로 맵 이동 (8px 임계값으로 탭/드래그 구분)
- **UI 카메라 분리**: 메인 카메라(줌/스크롤)와 UI 카메라(고정) 분리
  - 턴 정보, 턴 종료 버튼, 게임오버 텍스트, 액션 메뉴 → UI 카메라
- **자동 포커스**: 유닛 선택, 이동, AI 턴, 아군 턴 시작 시 카메라 팬
- **시작 포커싱**: 아군 유닛 평균 위치로 초기 카메라 설정
- **viewport**: `100dvh`, `touch-action: none`, `user-scalable=no`

### 6. 사운드/BGM 추가
- AudioManager: Web Audio API 프로시저럴 오디오 (외부 파일 없음)
- **BGM 4종**: 타이틀(D단조), 월드맵(C장조), 전투(A단조), 대화(앰비언트)
- **SFX 11종**: attack_hit, skill_cast, skill_heal, skill_fire, move, level_up, unit_die, game_over_win, game_over_lose, menu_click, turn_start
- 첫 터치 시 AudioContext 잠금 해제 (텔레그램 호환)
- 타이틀 화면 음소거 토글 버튼

### 7. 기술적 이슈 및 해결
- Railway `${{Postgres.XXX}}` 변수 참조 실패 → 실제 값 직접 입력
- Vercel Deployment Protection → OFF 필요
- 텔레그램 Menu Button에서 initData 미전달 → `/newapp`으로 Web App 등록
- `setScrollFactor(0)` + 카메라 줌 시 UI 위치 밀림 → UI 전용 카메라 분리
- 액션 메뉴 `_menuClickConsumed` 플래그 pointerdown에서 소비됨 → pointerup까지 유지
