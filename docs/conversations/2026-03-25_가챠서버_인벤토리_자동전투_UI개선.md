# 가챠 서버 이전, 인벤토리, 자동전투, UI 개선

## 날짜: 2026-03-25

## 주요 작업 내역

### 1. 가챠 시스템 서버 이전 (해킹 방지)
- 서버 GachaModule 생성 (Controller + Service)
- POST /gacha/pull: 서버에서 확률 계산, 재화 차감, 장수 추가
- GET /gacha/status: 보석/금화/천장 조회
- SaveEntity에 gems(기본 1000), gachaPity 필드 추가
- 클라이언트는 서버 API 호출만 (로컬 가챠 로직 제거)
- 세이브 자동 생성 시 기본 장수(여포/장료/고순) 포함

### 2. 등급 체계 변경
- B/A/S/SR/SSR → N/R/SR/SSR/UR
- 장수 관리에 등급 색상 표시

### 3. 시나리오 출전 시스템
- isScenarioUnit 플래그로 시나리오/가챠 장수 구분
- 시나리오: 고정 장수 + 게스트 1명 (가챠 장수 선택)
- 게스트 선택 UI (WorldMapScene)
- 모든 시나리오 맵에 4번째 시작 위치 추가

### 4. 인벤토리 시스템
- CampaignProgress에 equipmentBag, skillBag, materialBag 추가
- 로비 → 🎒 인벤토리 메뉴 (장비/스킬/소재 탭)
- 장착: 인벤토리 → 아이템 선택 → 장수 선택 → 장착
- 해제: 장수 상세 → 해제 → 인벤토리로 이동
- 초기 아이템: 청동검, 가죽갑, 전고, 격려, 방어강화, 회복

### 5. 자동 전투 / 배속
- AUTO 토글 버튼 (화면 좌상단)
- 1x/2x/3x 배속 토글
- 자동 전투 AI: 가장 가까운 적에게 접근 → 공격/스킬/대기
- 자동 모드 시 액션 메뉴 스킵 (직접 행동)
- 아군 턴 시작 시 자동 모드 자동 재개
- scrollFactor(0) + 줌 보정으로 버튼 표시 (UI 카메라 이슈 해결)

### 6. UI 개선
- React UnitInfoPanel 제거 (Phaser 내 패널로 대체)
- 액션 메뉴 표시 시 유닛 정보 패널 자동 숨김
- 가챠 장수 초기 스킬: 병종 기본 스킬만 (장착 스킬 없이 시작)
- 일반 가챠(금화) + 프리미엄 가챠(보석) UI 모두 구현

### 7. 기술적 이슈
- UI 카메라에서 AUTO/배속 버튼이 안 보이는 문제 → create() 마지막에 생성 + scrollFactor(0)
- 자동 전투 시 액션 메뉴에서 멈추는 문제 → onMoveComplete에서 autoMode 체크
- 가챠 장수에 불필요한 장착 스킬 포함 → 모두 빈 배열로 수정
