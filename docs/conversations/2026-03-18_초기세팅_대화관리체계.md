## 논의 내용

### 대화 관리 체계
- 3가지 조합: CLAUDE.md + 메모리 시스템 + 대화 로그

### 프로젝트 기획
- 1998년 KOEI 삼국지 조조전(SRPG)을 웹/텔레그램 미니앱으로 재현
- 기술 스택: Phaser.js + React + TypeScript + Vite
- 6단계 로드맵 수립

### Phase 1 프로토타입 구현
- Vite 5 + React + TypeScript + Phaser.js 프로젝트 스캐폴딩
- 공유 타입 정의 (Position, TileType, UnitData, BattleState)
- 12x10 사각 그리드 맵 렌더링 (평지, 숲, 산, 물 지형)
- 아군 3명 (조조, 하후돈, 하후연) / 적군 3명 (여포, 장료, 진궁)
- BFS 이동 범위 계산 + A* 경로 탐색
- 전투 시스템 (데미지 = ATK - DEF * 지형보정)
- 턴 시스템 (아군/적군 턴 교대)
- 적 AI (가장 가까운 아군 추적 + 공격)
- HP 바, 데미지 텍스트, 사망 처리
- 승리/패배 판정
- React UI (유닛 정보 패널, 턴 표시)
- 빌드 및 타입 체크 통과 확인

## 기술적 이슈
- Vite 8이 Node.js 20.18.0과 호환 불가 → Vite 5로 다운그레이드
- erasableSyntaxOnly가 enum/constructor parameter properties와 충돌 → 비활성화

## 결정사항
- Vite 5 사용 (Node 20.18 호환)
- Phaser는 게임 렌더링만, React는 오버레이 UI만 담당
- EventBus로 Phaser ↔ React 통신
- 게임 로직(GridSystem, CombatSystem, TurnSystem)은 Phaser 의존성 없는 순수 TS 클래스
