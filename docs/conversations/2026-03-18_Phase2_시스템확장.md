# 2026-03-18 | Phase 2 - 게임 시스템 확장 구현

## 구현 완료 내용

### Step 1: 병종 시스템
- UnitClass enum (보병/기병/궁병/책사/도적/무도가)
- 병종별 지형 이동비용 차등 (기병: 숲 3, 도적: 산 통과 가능 등)
- shared/types/unitClass.ts, shared/data/unitClassDefs.ts

### Step 2: 전투 시스템 강화
- 반격 시스템: 방어자 생존 + 사거리 내이면 50% 반격
- getEffectiveStats(): base + 상태효과 + 장비 통합
- CombatResult 확장 (counterDamage, attackerDied)

### Step 3: 계략(스킬) 시스템
- 10개 스킬: 화계, 수계, 회복, 대회복, 독계, 혼란, 격려, 방어강화, 화살비, 돌격
- SkillSystem: 스킬 실행, 상태효과 처리, 쿨다운
- BattleScene FSM에 AWAITING_ACTION, SKILL_TARGETING 상태 추가
- 액션 메뉴 (공격/스킬/대기)
- AI 스킬 사용 (힐 > AoE > 단일공격 우선)

### Step 4: 경험치 & 레벨업
- ExperienceSystem: 공격30 + 킬30 + 힐20 + 스킬25 EXP
- 100 EXP당 레벨업, 병종별 성장률 적용
- 레벨업 시 HP/MP 전체 회복

### Step 5: 장비 시스템
- 무기/방어구/악세서리 3슬롯
- 12개 장비: 청동검, 강철검, 철창, 장궁, 현자의 지팡이, 가죽갑, 철갑, 현자의 로브, 적토마, 전고, 옥새
- 병종 제한 장비 (장궁→궁병, 적토마→기병 등)

### Step 6: 통합
- 턴 시작 시 상태효과 틱 (독 데미지, 기절/혼란 스킵)
- 전체 빌드 및 타입 체크 통과
